## Phase A：上线准备（P0 阻塞清零）

目标：使现有安装包达到 Go-Live 交付标准。

### A-1 库存余额一致性修复（最高优先级，阻塞上线）

问题描述：`audit_inventory_balance_consistency.py` 报告 `findings=10`。
受影响物料：product_id 26（GT-MAT-TRIAL-001）、product_id 27（GT-MAT-TRIAL-002）。
受影响追溯维度：序列号 SN-GT-TRIAL-20260526-001、项目 PJ-GT-TRIAL-20260526-001。

执行步骤：

1. 备份数据库

```cmd
.venv\Scripts\python.exe scripts\pg_backup.py --output backups\pre_inventory_fix.dump
```

2. 执行 dry-run 确认修复范围

```cmd
set PG_PASSWORD=admin && .venv\Scripts\python.exe scripts\repair_inventory_balance_consistency.py --dry-run
```

3. 业务方审核修复清单并书面确认
4. 执行修复（使用已备好的 execute_fix_inline.py）
5. 重新运行一致性审计，目标 findings=0

```cmd
set PG_PASSWORD=admin && .venv\Scripts\python.exe scripts\audit_inventory_balance_consistency.py
```

验收标准：`findings=0`，全套基线审计重新通过。

## Phase B：P0 引擎深化（上线后第 1-8 周）

路由已在 `MENU_ROLLOUT_CLASSIFICATION.md` 定义为 live，需要后端实现深化。

### B-1 MRP / Kitting 净需求引擎

业务价值：从销售订单/项目自动展开采购和生产建议，消除人工排单盲区。

| 任务 | 说明 |
|---|---|
| MRP 展开算法 | 基于 BOM + 库存余额 + 在途数量计算净需求 |
| 建议单转换 | MRP suggestion 转采购申请 / 工单 / 外协订单，写入 trace_link |
| Kitting 缺料看板 | 按项目/序列号/工单显示覆盖率、缺口量、采购/生产建议 |
| 运行历史 | MRP 运行快照持久化，支持对比前后差异 |

关键约束：MRP 不直接创建库存事务和财务凭证；仅生成建议单，由操作员确认后转换。

需新增数据库表（先写入 `services\schema_migrations.py`）：
- `mrp_runs`：MRP 运行记录
- `mrp_run_items`：MRP 展开明细
- `mrp_suggestions`：建议单

---

### B-2 项目/序列号全链追溯引擎

业务价值：一台机器从报价到采购、生产、发货、售后的完整履历可查。

| 任务 | 说明 |
|---|---|
| 追溯图谱 | /trace/project/ 和 /trace/serial/ 实现前向/后向追溯图 |
| 文档追溯 | /trace/document// 从任意单据展开上下游关联 |
| 完整性扫描 | /trace/integrity/scan 写入缺失链接发现，支持逐条修复 |
| 追溯完整性得分 | 按项目维度评估采购/生产/发货/服务各环节覆盖率 |

当前风险：`require_project_serial=0`，追溯字段非强制。建议上线稳定后逐步开启强制校验。

---

### B-3 单机/项目成本归集引擎

业务价值：核算每台机器（序列号）的实际制造成本，支撑定价和利润分析。

| 任务 | 说明 |
|---|---|
| 成本归集运行 | /cost/run 按期间+项目+序列号归集材料/人工/制造费用/外协/服务/质检成本 |
| 成本构成报表 | /cost/project/ 和 /cost/serial/ 只读汇总视图 |
| 成本对账 | 业务成本 vs 库存成本 vs 总账成本三维比对 |
| 差异分析 | 标准成本 vs 实际成本差异（为后期预算控制做基础）|

关键约束：成本归集不修改单据状态，不生成财务凭证，仅提供汇总只读视图。

---

### B-4 BOM/图纸/工艺/ECN 快照与影响控制

业务价值：锁定工单发料时的 BOM 版本，防止设计变更影响在制品。

| 任务 | 说明 |
|---|---|
| BOM 版本状态机 | 草稿 -> 审批 -> 发布 -> 废止，工单释放时冻结版本快照 |
| BOM 替代料管理 | 审批替代料，支持自动替换标志和替换比例 |
| ECN 影响分析 | 分析变更影响的采购单/工单/外协单/库存/图纸 |
| ECN 行动任务 | 自动生成采购变更/工单变更/领料调整/图纸替换任务并跟踪关闭 |
| 工单 BOM 快照 | /work-orders//snapshots 展示工单发布时冻结的 BOM/工艺/图纸版本 |

---

### B-5 数据级权限控制

业务价值：按客户/供应商/项目/仓库维度隔离不同销售员/采购员的可见数据范围。

| 任务 | 说明 |
|---|---|
| 权限规则 CRUD | /security/data-permissions 增删改查规则 |
| 查询范围注入 | 列表查询自动附加 data_scope_rules 过滤条件 |
| 访问日志 | /security/data-access-logs 记录查看/导出/编辑访问决策 |
| 导出审批 | /security/export-approvals 敏感导出需要管理员审批 |

注意：此功能影响所有列表查询性能，上线前必须完成全量权限/导航审计。

## Phase C：财务 Phase 2（上线后第 5-12 周）

| 任务编号 | 任务名称 | 优先级 | 说明 |
|---|---|---|---|
| C-1 | 凭证受控生成 | P1 | 从销售发票/采购发票/收付款单生成标准凭证；操作员确认后过账 |
| C-2 | 银行对账单导入 | P1 | 导入银行 CSV/Excel；自动匹配现金银行日记账；人工勾选未匹配项 |
| C-3 | 期末关账规则完善 | P1 | 完善关账前置检查：未审批凭证、未匹配银行项、未关闭发票；期间关账后锁定历史数据修改 |
| C-4 | 多行资金日记账完善 | P2 | 一张收付款单多笔资金账户的日记账拆分 |
| C-5 | 外汇期末调整 | P2 | 汇率表维护；外币 AR/AP 按期末汇率调整差额 |
| C-6 | 法定财务报表 | P3 | 资产负债表、利润表、现金流量表（基于科目余额汇总）|
| C-7 | 坏账计提 | P3 | 按账龄比例计算坏账准备，生成调整凭证 |

## Phase D：运营质量提升（穿插进行）

### D-1 质量管理深化

- 来料检验结果关联采购收货单（合格/不合格分批入库）
- 完工检验结果关联工单（影响完工入库数量）
- 服务维修质检关联 RMA（影响返厂/报废决策）
- 质检报告支持打印和导出

### D-2 生产排程优化

- 工单工序排程看板（按工作中心甘特图式显示）
- 工序开始/完工报工（操作员扫码或手工录入）
- 工作中心产能日历（设置每日可用工时）
- 排程冲突预警（超产能自动告警）

### D-3 文档附件与打印优化

- 合同/图纸/检验报告附件绑定到业务单据
- 打印模板维护支持更多单据类型
- 批量导出（按日期/客户/项目范围的 Excel 导出）

### D-4 移动端/响应式适配

- 仓库人员移动端扫码入库/出库（响应式页面优化）
- 生产报工移动端界面（工序报工简化表单）

## 五、关键风险与管控措施

| 风险 | 影响程度 | 管控措施 |
|---|---|---|
| 库存余额一致性未修复强行上线 | 高 | 必须在上线前修复至 findings=0，此项为硬阻塞 |
| MRP/成本引擎引入新过账逻辑 | 高 | 严格遵守"引擎只生成建议，不直接过账库存或财务"原则 |
| BOM 版本控制与在制工单冲突 | 中 | 工单释放时立即冻结 BOM 快照，不允许事后修改已用版本 |
| 数据级权限改造影响所有列表查询 | 中 | 全量导航和直接访问矩阵审计，灰度上线逐步验证 |
| 财务凭证生成逻辑错误 | 高 | 凭证生成必须经操作员确认，批量生成必须有预览对比和撤销机制 |
| 中文文本变更引入乱码 | 中 | 所有中文文本变更经 source_integrity_audit.py 验证后合入 |
| 大表无索引导致报表超时 | 中 | 每个新增查询必须有执行计划评审，关键字段建立索引 |
