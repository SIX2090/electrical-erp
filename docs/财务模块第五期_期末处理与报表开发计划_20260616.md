# Finance Phase 5 Development Plan: Period Close and Financial Statements

## Boundary

This document records the existing Phase 5 finance integration boundary for period close and financial statement reporting. It does not introduce new modules, routes, database fields, menus, or accounting rules.

## Business Loop

- Source documents: posted inventory movements, receivables, payables, vouchers, settlement records, and cost records.
- Target outputs: period close checks, period close snapshots, balance sheet, income statement, and cash flow statement.
- Data owner: finance.
- Upstream owners: sales, purchase, warehouse, production, outsourcing, and service teams.
- Downstream impact: closed periods must support reconciliation and reporting without changing historical source documents.

## Acceptance Checks

- Period close routes and financial report routes are registered in the application.
- Period close templates and financial statement templates render through the normal ERP shell.
- Navigation exposes period close and financial report entries only in the finance/system-controlled surface.
- Reports remain read-only except for refresh, export, print, and drill-down.
- Period close does not add statutory accounting logic beyond the existing implemented service boundary.

## Verification

The integration is verified by `scripts/verify_phase5_integration.py` and the normal ERP release audits. Any future accounting-rule change must define a separate written boundary before code changes.
